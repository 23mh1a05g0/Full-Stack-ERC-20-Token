# Full-Stack ERC-20 Token Faucet DApp

A production-ready decentralized application (DApp) that distributes ERC-20 tokens with enforced rate limits — built with Solidity, Hardhat v3, React, ethers.js, and Docker.

## Architecture

```
.
├── contracts/
│   ├── Token.sol          # ERC-20 token (YourToken / FTK)
│   ├── TokenFaucet.sol    # Rate-limited faucet
│   └── test/
│       └── TokenFaucet.test.js  # 19 Hardhat/Mocha unit tests
├── scripts/
│   └── deploy.js          # Automated deployment + ABI export
├── frontend/
│   ├── src/
│   │   ├── App.jsx        # React UI with wallet connection & countdown
│   │   ├── index.css      # Premium dark-mode design system
│   │   ├── main.jsx       # React entry point
│   │   └── utils/
│   │       ├── contracts.js   # ethers.js contract helpers
│   │       ├── contractsData.json  # Generated by deploy.js
│   │       └── eval.js        # window.__EVAL__ interface
│   ├── server.js          # Express server + /health endpoint
│   ├── Dockerfile         # Multi-stage Docker build
│   └── package.json
├── hardhat.config.js
├── docker-compose.yml
└── .env.example
```

## Smart Contracts

| Contract | Description |
|---|---|
| `YourToken` (FTK) | ERC-20 with `MAX_SUPPLY` of 1 billion. Only the designated `minter` can call `mint()`. |
| `TokenFaucet` | Distributes 100 FTK per claim with a 24-hour cooldown and 1,000 FTK lifetime cap per address. Admin can pause/unpause. |

### Key Parameters

| Parameter | Value |
|---|---|
| Faucet amount per claim | 100 FTK |
| Cooldown period | 24 hours |
| Lifetime limit per address | 1,000 FTK |
| Token max supply | 1,000,000,000 FTK |


| Variable | Description |
|---|---|
| `SEPOLIA_RPC_URL` | Alchemy / Infura Sepolia endpoint |

| `ETHERSCAN_API_KEY` | For automatic contract verification |

## Getting Started

### 1. Install dependencies

```bash
npm install
cd frontend && npm install && cd ..
```

### 2. Compile contracts

```bash
npx hardhat compile
```

### 3. Run tests (19 passing)

```bash
npx hardhat test mocha -- "contracts/test/TokenFaucet.test.js"
```

### 4. Deploy locally

```bash
# Terminal 1 – start local node
npx hardhat node

# Terminal 2 – deploy + export ABIs
npx hardhat run scripts/deploy.js --network localhost
```

### 5. Deploy to Sepolia

```bash
npx hardhat run scripts/deploy.js --network sepolia
```

### 6. Run the frontend (development)

```bash
cd frontend
npm run dev
```

### 7. Run with Docker

```bash
# Build and start
docker compose up --build

# Health check
curl http://localhost:3000/health
```

## Evaluation Interface (`window.__EVAL__`)

The frontend exposes a global object for automated testing:

```javascript
// Connect wallet
const address = await window.__EVAL__.connectWallet();

// Request tokens
const receipt = await window.__EVAL__.requestTokens();

// Query state
const balance   = await window.__EVAL__.getBalance("0x...");
const eligible  = await window.__EVAL__.canClaim("0x...");
const remaining = await window.__EVAL__.getRemainingAllowance("0x...");
const addrs     = window.__EVAL__.getContractAddresses();
// → { tokenAddress, faucetAddress }
```

## Testing

The test suite covers all critical paths:

1. Token deployment and initial state
2. Faucet deployment and configuration
3. Successful token claim and event emission
4. Cooldown enforcement (immediate re-claim, 23h59m, exactly 24h)
5. Lifetime limit enforcement (10 claims = 1,000 FTK)
6. Pause mechanism (admin-only, blocks claims, resumes after unpause)
7. Insufficient supply edge case
8. Independent user cooldown tracking

## Security Practices

- **Checks-Effects-Interactions** pattern in `requestTokens()`
- **Owner-restricted** minting via `setMinter()` — only the faucet address can mint
- **Admin-only** pause mechanism
- **MAX_SUPPLY** enforced on-chain in `mint()`
- No ETH handling — token-only transfers
